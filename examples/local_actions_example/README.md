# LocalActions example (sdk-js)

Demo of the standalone `LocalActions` class talking to kwild's admin port for
tn_local stream operations. No chain id or `NodeTNClient` required — the
admin server handles its own transport auth, and tn_local derives the
`data_provider` server-side from the node's secp256k1 key.

Optionally signs every admin request with the operator key so the example
works against nodes that have `[extensions.tn_local] require_signature = true`.

## Setup

1. Build and start the node with the **admin opt-in** task (default
   `single:start` keeps admin on a unix socket inside the container, which
   is unreachable from the host):

   ```bash
   cd /path/to/trufnetwork/node
   task docker:build:local      # first run, or after node code changes
   task single:start:admin      # exposes admin RPC on 127.0.0.1:8485
   ```

   `single:start:admin` layers `compose.admin.yaml` on top of the default
   compose file. It binds host port `8485` to **`127.0.0.1` only** and
   passes `--admin.listen 0.0.0.0:8485 --admin.notls` inside the container.
   Convenient for local development, but **not equivalent to the default
   unix socket**: any process running as any user on the host can reach
   loopback TCP, while the unix socket gates access by filesystem
   permissions.

   **For production, do not use plaintext HTTP on the admin port.** Pick
   one of:
   - the default unix socket (`single:start` without the `:admin` task),
   - mTLS (`--admin.tls.*` plus client certs), or
   - `--admin.pass` (admin password).

   For multi-host setups (SDK and node on different machines), set up
   mTLS or `--admin.pass` first and only then override `TN_ADMIN_BIND=0.0.0.0`
   to expose the port off-loopback. See `node/docs/development.md` and
   `node/compose.admin.yaml` for the full options.

2. Build sdk-js so the example can import the local package:

   ```bash
   cd /path/to/trufnetwork/sdk-js
   pnpm install
   pnpm run build
   ```

3. Install the example deps:

   ```bash
   cd /path/to/trufnetwork/sdk-js/examples
   npm install
   ```

## Run — unsigned (node has `require_signature = false`)

```bash
cd /path/to/trufnetwork/sdk-js/examples
npx tsx local_actions_example/main.ts
```

## Run — signed (node has `require_signature = true`)

Start the node with the env var that layers `require_signature = "true"` into
`config.toml`:

```bash
cd /path/to/trufnetwork/node
TN_LOCAL_REQUIRE_SIGNATURE=true task single:start:admin
```

Export the autogenerated operator key into your environment, then run the
example. Exporting (vs. passing the key on the command line) keeps the
private key out of `argv` — anyone on the host can read it from `ps aux`,
shell history, or container logs otherwise.

```bash
cd /path/to/trufnetwork/sdk-js/examples

export OPERATOR_KEY="$(docker exec tn-db cat /root/.kwild/nodekey.json | jq -r '.key')"
npx tsx local_actions_example/main.ts
```

The example first signs a request with a freshly generated (wrong) key and
expects the server to reject it — a positive demonstration that admin port
reachability alone is no longer enough to act as the operator — then re-runs
the full flow with the real operator key.

## What this demonstrates

- `new LocalActions(new AdminClient({ adminProvider }), { signer })` —
  optional operator-key signer, opt-in `_auth` envelope on every admin call.
- Wrong-key rejection when `require_signature = true` — admin-port
  reachability alone does not grant operator privileges.
- `createStream` / `insertRecords` / `getRecord` / `getIndex` — primitive
  workflow with parallel-array `insertRecords` input.
- `insertTaxonomy` + `disableTaxonomy` — composed stream lifecycle without
  consensus.
- `listStreams` — every row is owned by the node, `dataProvider` is the
  node's own lowercased address.
- `deleteStream` — cleanup; safe to call on a missing stream (we wrap in
  try/catch in the example).

For projects that already use a full `NodeTNClient` (gateway-backed, needs a
signer), call `client.loadLocalActions({ adminProvider }, { signer })` to
share the admin transport with the rest of the app.
